# Security policy

Markagent is a Chrome extension that runs locally in your browser. It has no servers, no telemetry, and no data leaves your machine. That said, browser extensions are powerful — if you find a security issue, please report it responsibly.

## Reporting a vulnerability

**Do not** open a public GitHub issue for security problems.

Instead, email: `tunghcltt.work@gmail.com` with:

- A clear description of the vulnerability
- Steps to reproduce
- Affected Markagent version
- Your browser and version
- Any proof-of-concept code or screenshots (optional)

You'll get a response within 72 hours. Confirmed issues are patched and shipped to the Chrome Web Store as soon as possible.

## Scope

In scope:

- The Markagent Chrome extension (current version on the Web Store)
- The Markagent landing site at `markagent.avibebuilder.com`

Out of scope:

- Third-party AI coding agents (Claude Code, Cursor, etc.) — report to them directly
- Social engineering, physical access attacks
- Denial of service against the landing page

## Disclosure

Once a fix ships, the vulnerability is disclosed in the [release notes](../../releases) with credit to the reporter (unless you'd prefer to stay anonymous).

Thanks for keeping Markagent users safe.
